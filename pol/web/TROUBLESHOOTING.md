# Решение проблем: не открывается на телефоне

## Проблема: адрес http://192.168.0.105:8080 не открывается на телефоне

### Проверка 1: Сервер запущен правильно?

В окне `start_server.bat` должно быть написано:
```
Сервер запущен. Закройте окно для остановки.
```

Если там ошибка про Python — установите Python с [python.org](https://www.python.org/downloads/).

### Проверка 2: ПК и телефон в одной Wi‑Fi сети?

- На ПК: откройте **Настройки → Сеть и Интернет → Wi‑Fi** — посмотрите название сети.
- На телефоне: **Настройки → Wi‑Fi** — должно быть **то же самое** название сети.

Если сети разные — подключите телефон к той же Wi‑Fi, что и ПК.

### Проверка 3: Файрвол Windows блокирует подключение

1. На ПК откройте **Панель управления → Система и безопасность → Брандмауэр Защитника Windows**.
2. Нажмите **Дополнительные параметры** (слева).
3. Выберите **Правила для входящих подключений** → **Создать правило**.
4. Тип правила: **Для порта** → Далее.
5. Протокол: **TCP**, Удалённые порты: **8080** → Далее.
6. Действие: **Разрешить подключение** → Далее.
7. Профили: отметьте все (Домен, Частная, Публичная) → Далее.
8. Имя: "Python HTTP Server 8080" → Готово.

Или проще: в окне `start_server.bat` нажмите правой кнопкой → **Запуск от имени администратора** (может помочь).

### Проверка 4: Правильный IP адрес

В окне `start_server.bat` показан IP. Проверьте, что он правильный:

1. На ПК откройте **Командную строку** (Win+R → `cmd` → Enter).
2. Введите: `ipconfig`
3. Найдите раздел **"Адаптер беспроводной локальной сети Wi‑Fi"** (или похожий).
4. Посмотрите строку **"IPv4-адрес"** — должно совпадать с тем, что в окне сервера.

Если не совпадает — используйте IP из `ipconfig`.

### Проверка 5: Работает ли на самом ПК?

На **том же компьютере**, где запущен сервер, откройте браузер и введите:
```
http://localhost:8080
```

Если открывается — сервер работает. Если нет — проблема в Python или порте.

### Альтернатива: использовать другой порт

Если порт 8080 занят или блокируется, можно изменить порт:

1. Откройте `start_server.bat` в блокноте.
2. Замените все `8080` на другой порт, например `3000` или `5000`.
3. Сохраните и запустите снова.
4. На телефоне используйте новый порт: `http://192.168.0.105:3000`

### Альтернатива: использовать Node.js вместо Python

Если Python не работает:

1. Установите [Node.js](https://nodejs.org/) (если ещё нет).
2. В папке `web` откройте командную строку и выполните:
   ```
   npx serve . -p 8080 --listen 0.0.0.0:8080
   ```
3. Используйте адрес, который покажет `serve`.

### Быстрая проверка: ping с телефона

На телефоне установите приложение для ping (например, "Network Tools" или "Fing") и проверьте, доступен ли ПК:
- Введите IP `192.168.0.105` и нажмите ping.
- Если пакеты доходят — сеть работает, проблема в сервере или файрволе.
- Если не доходят — ПК и телефон в разных сетях или файрвол блокирует всё.
